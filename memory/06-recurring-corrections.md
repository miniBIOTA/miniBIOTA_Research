# Recurring Corrections

Use this file to avoid known mistakes.

## Do Not Use Test Writes

Mistake: creating dummy Supabase records to inspect a function return value.

Correct behavior: read source, signatures, schema, or documentation instead.

Why it matters: test writes create real ecological records and can persist if not caught.

## Do Not Mix Story Threads And Open Loops

Mistake: treating every new ecological development as an open loop.

Correct behavior: story threads are internal tracking; open loops are audience-facing questions from published content.

Why it matters: mixing them corrupts both research tracking and public content continuity.

## Do Not Upgrade Uncertainty For Narrative Clarity

Mistake: turning `Possible` into `Likely`, `Likely` into `Confirmed`, or unresolved recruitment into establishment.

Correct behavior: keep the confidence label matched to the evidence.

Why it matters: public-facing claims and downstream records depend on conservative scientific boundaries.

## Do Not Treat Migrated Docs As Live Data Without Checks

Mistake: using local overview counts as current species, observation, or thread totals.

Correct behavior: verify Supabase when current structured state matters.

Why it matters: miniBIOTA records change quickly.
